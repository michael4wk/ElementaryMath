# 下一阶段质量优化方案（工程化增强版）

## 1. 文档定位

本方案用于指导“阶段性交付完成后”的质量提升工作，目标是把当前“可运行、可验收”的系统升级为“高一致性、低缺失、可持续治理”的生产级知识库能力。  
与现有建议相比，本方案不只做指标拉升，还补齐质量治理闭环（标准 → 执行 → 监控 → 纠偏 → 发布门禁）。

## 2. 当前基线（以现有验收结果为准）

- 题干-答案-解析一致率：79.51%
- 解析缺失率：19.26%
- 易错点缺失率：35.75%
- 抽取成功率：100.0%（流程稳定，但质量仍有提升空间）

结论：当前主要矛盾不是“能不能跑”，而是“结果是否稳定可信、是否可规模化演进”。

## 3. 目标体系（新增工程化目标）

## 3.1 北极星目标

- 构建“可量化、可追责、可回滚”的质量治理体系，保证每次增量接入后质量不回退。

## 3.2 质量目标（下一阶段）

- 一致率：题干-答案-解析一致率 ≥ 92%
- 完整性：解析缺失率 ≤ 8%，易错点缺失率 ≤ 15%
- 稳定性：增量发布后关键质量指标 0 回退（较前一版本不下降）
- 可追溯：高优先问题 100% 可定位到 source_ref 和处理责任链路

## 3.3 服务级目标（SLO）

- 质量报表日级产出成功率 ≥ 99%
- 质量门禁判定任务成功率 ≥ 99%
- 回归校验任务失败可在一个发布窗口内完成重试与复核

## 4. 设计原则（按工程最佳实践补强）

- 质量前置：把“发布后发现问题”改为“发布前门禁拦截”
- 指标分层：全局指标 + 年级维度 + 题型维度，避免平均值掩盖问题
- 根因驱动：每个质量问题都要求“症状-根因-修复动作”三段式闭环
- 变更可控：所有策略升级采用灰度与可回滚机制
- 人机协同：规则/模型自动化优先，人工只聚焦高价值疑难样本
- 成本意识：优先改造高收益、可复用链路，避免一次性重写

## 5. 问题拆解与根因假设

## 5.1 一致率不足（79.51%）

可能根因：
- 抽取后字段对齐策略弱，未形成“题干-答案-解析”的强约束校验
- 解析文本存在跨段落或格式噪声，导致语义映射偏移
- 缺少冲突类型分级（数值冲突、单位冲突、步骤冲突）

治理方向：
- 增加一致性规则引擎（符号/单位/步骤结构校验）
- 引入语义一致性二检（规则 + 模型双通道）
- 建立冲突问题分级队列与自动回灌机制

## 5.2 解析缺失（19.26%）

可能根因：
- 源文档解析段落识别规则不稳定
- 结构化阶段未区分“无解析”与“解析抽取失败”
- 缺少按来源与章节的缺失热区定位

治理方向：
- 扩展段落模板与章节锚点识别规则
- 增加“缺失原因码”字段（源无/抽取失败/清洗丢失）
- 对高频缺失热区执行专项修复

## 5.3 易错点缺失（35.75%）

可能根因：
- 易错点抽取依赖隐式表达，规则召回不足
- 模型抽取缺少领域词典与年级语义约束
- 缺少“题型→易错点模板”映射

治理方向：
- 建立小学数学错因词典与题型模板库
- 引入弱监督标注集，持续优化抽取提示与规则
- 建立低置信样本人工复核池，回流训练与规则迭代

## 6. 目标架构（质量治理闭环）

## 6.1 新增能力模块

- 数据契约层：定义必填字段、枚举值、跨字段一致性约束
- 质量规则引擎：完整性/一致性/可追溯性规则统一执行
- 问题分诊中心：按严重级别、根因类型、责任模块自动分桶
- 质量门禁层：发布前强制校验，不达标则阻断发布
- 质量观测层：日/周趋势看板、回退告警、变更影响分析

## 6.2 发布门禁策略

- 硬门禁（必须通过）：一致率、解析缺失率、可追溯完整率
- 软门禁（允许发布但需工单）：易错点覆盖率、年级分层异常项
- 兜底机制：出现门禁失败时自动触发回滚候选版本与问题清单

## 7. 分阶段落地方案（可执行）

## 阶段 P0：标准与基线固化

- 固化指标口径、统计脚本、责任人映射（避免口径漂移）
- 建立黄金样本集（按年级/题型分层）作为回归基准
- 完成缺失原因码与冲突类型码设计

交付物：
- 质量指标字典
- 黄金样本集与基准报告
- 问题分类与责任映射表

## 阶段 P1：一致率专项提升

- 落地一致性规则引擎（数值、单位、步骤逻辑）
- 增加语义二检与冲突置信度评分
- 建立“冲突自动修复 + 人工复核”双路径

交付物：
- 一致性规则包（可版本化）
- 冲突问题清单与修复闭环报表
- 一致率专项周报

## 阶段 P2：缺失率专项治理

- 解析段落识别规则增强与模板扩展
- 易错点词典、题型模板、抽取策略迭代
- 缺失热区治理与专项回填

交付物：
- 解析识别增强规则包
- 易错点抽取模板库
- 缺失治理报告（按年级/题型）

## 阶段 P3：门禁化与持续运营

- 将质量校验接入发布流水线，形成硬/软门禁
- 建立趋势看板与告警策略
- 建立月度复盘机制（问题复发率、修复时效、回退率）

交付物：
- 门禁策略配置与执行记录
- 质量趋势看板定义
- 复盘模板与运营SOP

## 8. 任务分解（执行清单）

- 建立质量指标字典与统计口径版本管理
- 构建分层黄金样本并接入回归校验任务
- 增加一致性校验规则与语义二检链路
- 建立冲突分级、修复、复核、回灌流程
- 增加缺失原因码并输出缺失热区分析
- 建立易错点词典与题型模板抽取策略
- 将关键指标接入发布门禁与回滚触发器
- 建立质量趋势看板与异常告警策略
- 建立跨角色责任机制（数据、算法、业务、运维）

## 9. 验收标准（阶段结束判定）

- 指标达标：一致率、解析缺失率、易错点缺失率达到目标阈值
- 稳定性达标：连续两个发布周期无关键指标回退
- 闭环达标：高优先问题具备完整工单链路与复盘结论
- 运营达标：门禁、看板、告警、回滚机制均可演练通过

## 10. 风险与应对

- 规则过拟合风险：采用黄金样本 + 留出集双评估
- 模型波动风险：固定模型版本并实施灰度发布
- 人工负担上升风险：仅把低置信高影响样本送人工复核
- 指标“好看但无业务价值”风险：增加按年级/题型分层考核
- 发布阻塞风险：软硬门禁分层，确保治理与交付平衡

## 11. 组织与协作建议

- 设立质量Owner：统一口径、优先级与验收裁决
- 设立专项例会：一致率、缺失率、复发率三条主线跟踪
- 统一工单模板：问题现象、根因、修复动作、验证证据、回归结果
- 建立“变更即评估”机制：任何抽取策略变更必须附质量影响报告

## 12. 立即启动项（本周可执行）

- 冻结当前质量基线并生成首版指标字典
- 抽取并标注第一批黄金样本（覆盖主要年级与题型）
- 建立一致性冲突类型码与缺失原因码
- 启动一致率专项规则迭代的首轮验证
- 输出第一版“门禁前置”实施草案与回滚预案
