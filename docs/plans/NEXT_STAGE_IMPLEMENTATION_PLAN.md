# 下一阶段实施任务分解（优先级与责任角色）

## 1. 目标

- 在现有“质量治理闭环已可运行”的基础上，进入持续优化与规模化运营阶段。
- 将当前能力从“可用”提升到“高效率、低人工成本、可持续迭代”。

## 2. 角色分工

- 数据工程：抽取规则、热区修复、样本治理、数据回流
- 算法工程：语义一致性策略、置信度优化、模板学习
- 后端工程：门禁接口、差异报告、审计与追溯能力
- 运维工程：告警路由、发布回滚演练、稳定性巡检
- QA工程：验收门槛、回归用例、演练验证
- 产品/教研：问题优先级裁定、复核策略、业务口径确认

## 3. 工作流与优先级

## W1：高频缺失场景专项修复（P0）

- 目标：完成解析缺失热区的专项收敛，降低重复问题输入
- 主责：数据工程；协作：算法工程、QA
- 输入：`artifacts/quality/ops_report.json` 热点专题与冲突类型
- 任务：
  - 固化 Top20 热点专题修复清单
  - 对每个专题建立“规则修复 + 回归样本”双清单
  - 输出专项修复前后对比报告
- DoD：
  - 高频缺失场景完成一轮全量修复
  - 对应冲突项在下一周期显著下降

## W2：语义一致性置信度提升（P0）

- 目标：将语义一致性均值提升至可控区间，减少误报
- 主责：算法工程；协作：数据工程、QA
- 输入：`validation.consistency.semantic_consistency_confidence_avg_pct`
- 任务：
  - 按题型建立语义置信度阈值分层
  - 优化弱重叠判定策略，减少结构性误判
  - 引入复核反馈对阈值进行校准
- DoD：
  - 语义置信度均值达到阶段目标
  - 告警中 semantic_weak_overlap 占比下降

## W3：人工复核闭环提效（P1）

- 目标：提升复核关闭率，缩短问题从发现到闭环时间
- 主责：产品/教研；协作：数据工程、QA
- 输入：`review_pool.jsonl`、`review_backflow.json`
- 任务：
  - 定义复核SLA（P0/P1/P2 响应时限）
  - 建立批量复核模板与常见动作字典
  - 每周输出复核关闭率与积压趋势
- DoD：
  - 复核关闭率达到周度目标
  - 积压量连续下降

## W4：门禁策略分环境治理（P1）

- 目标：提升门禁策略精度，避免过严或过松
- 主责：后端工程；协作：运维、QA
- 输入：`config/quality_gate_profiles.json`
- 任务：
  - 建立 dev/staging/prod 阈值变更审计流程
  - 增加门禁阈值变更回放报告
  - 建立门禁异常快速降级预案
- DoD：
  - 阈值变更全量可审计
  - 演练场景下可快速恢复稳定门禁

## W5：运营看板与告警落地（P1）

- 目标：把报告变成可执行运营动作
- 主责：运维工程；协作：产品、QA
- 输入：`quality_trend.json`、`quality_alerts.json`、`quality_repair_tracking.json`
- 任务：
  - 定义告警分级与路由策略
  - 建立周会固定看板模板
  - 建立月度复盘结论沉淀规范
- DoD：
  - 告警有明确负责人和处理状态
  - 周度/月度运营节奏稳定执行

## W6：回滚与发布演练自动化（P2）

- 目标：降低人工演练成本，提高发布安全冗余
- 主责：运维工程；协作：后端、QA
- 输入：`rotation_actions.jsonl`、`release_diff.json`
- 任务：
  - 把演练步骤脚本化为一键流程
  - 输出演练前后一致性校验报告
  - 建立失败自动中止与恢复策略
- DoD：
  - 演练可重复执行
  - 失败场景可自动恢复到稳定态

## 4. 执行节奏建议

- Sprint 1（P0优先）：W1 + W2 启动并交付首轮结果
- Sprint 2（闭环提效）：W3 + W4 并行推进
- Sprint 3（运营固化）：W5 + W6 完成自动化与制度化

## 5. 关键依赖

- 复核反馈持续供给（`review_feedback.jsonl`）
- 质量历史稳定积累（`gate_history.jsonl`）
- 运营会议机制持续执行（周会/月会）

## 6. 风险与缓解

- 复核资源不足：优先处理 P0/P1 与低置信高影响样本
- 规则过拟合：保持留出集与跨题型校验
- 告警疲劳：调整阈值并按等级路由
- 变更过快：要求阈值变更必须附回放报告

## 7. 下一步启动清单（立即执行）

- 产出 Top20 热点专题修复任务单
- 设定语义置信度阶段目标与题型分层阈值
- 定义复核SLA并发布执行规范
- 建立门禁阈值变更审计模板
- 固化周报与月报模板
