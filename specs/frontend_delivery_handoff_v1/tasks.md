# 小学数学结构化知识库前端独立交付补齐任务清单（Tasks v1.1）

## 阶段 A：目录重组适配与资产落位（低冲突优先）

- 对齐 `docs/` 与 `specs/` 新目录约定，冻结交付资产落位规范
- 在 `docs/delivery/` 下创建 `api/` 子目录作为前端移交文档唯一入口
- 盘点现有可用接口并形成端点清册
- 输出 OpenAPI 3.x 契约初稿并完成字段语义对齐
- 建立统一错误码字典与标准错误对象定义
- 更新 `docs/delivery/FRONTEND_DELIVERY_READINESS.md` 的引用与口径
- 产出接口兼容映射表（现有路径到目标契约映射）

## 阶段 B：联调可用化（中等冲突）

- 明确 CORS 环境白名单与安全边界
- 补齐 `OPTIONS` 预检与跨域响应策略
- 产出前端接入手册（鉴权、分页、过滤、排序、错误处理）
- 产出环境矩阵文档（local/dev/staging/prod）
- 建立前端最小接口集示例请求/响应集合
- 提供 Mock 数据包与前端联调样例流程

## 阶段 C：一致性校验与发布协同（持续治理）

- 建立契约一致性校验流程并接入交付检查
- 建立接口变更评审流程与版本发布规则
- 建立兼容性检查项（新增字段、弃用字段、破坏性变更）
- 建立联调回归用例（成功、鉴权失败、参数错误、资源缺失）
- 建立变更公告模板与发布节奏
- 建立“文档口径优先级”规则（验收报告优先于历史总结）

## 阶段 D：高冲突改动后置实施（等待主线窗口）

- 评估并规划路径版本化（如 `/v1`）的迁移方案
- 评估并规划历史错误响应格式统一改造方案
- 评估并规划网关层鉴权与限流策略收口方案
- 在优化主线收敛窗口执行高风险改动并回归验证

## 跨阶段保障任务

- 建立“契约冻结窗口”机制，避免并行改动引发漂移
- 固化“新增优先、兼容优先”的变更原则
- 建立前后端周度对齐机制与差异清单
- 建立文档版本与服务版本双向追溯关系
- 建立问题工单模板，绑定责任人与回归结果
- 保证新增交付文档仅落位于 `docs/delivery/api/`，避免根目录扩散
